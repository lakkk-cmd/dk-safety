# 대경안심전기 내부 시연/테스트 가이드

이 문서는 내부 시연과 기능 검증을 빠르게 진행하기 위한 실무용 체크리스트입니다.

---

## 1) 실행 준비

1. `.env.example` 파일을 복사해 `.env.local` 생성
2. 아래 값 입력
   - `NEXT_PUBLIC_BUSINESS_PHONE`
   - `NEXT_PUBLIC_KAKAO_OPENCHAT_URL`
   - `ADMIN_PASSWORD`
3. 실행
   - `npm install`
   - `npm run dev`
4. 접속
   - `http://localhost:3000`

---

## 2) 10분 시연 동선 (발표용)

### A. 입주민 로그인 + 서비스 메인 (약 2분)

- 경로: `/` (입주민 로그인) → 성공 시 `/home` (서비스 메인)
- 확인 포인트
  - 제목: `우리집 전기 주치의 대경안심전기` 등 브랜드 메시지
  - 로그인 후 홈에서 서비스 카드, 예약·자가진단 CTA
  - 전화 문의, 카카오톡 상담(하단 퀵액션 포함)

멘트 예시:
"먼저 입주민을 식별하고, 홈에서 예약·자가진단·문의로 자연스럽게 이어지게 했습니다."

### A-1) (선택) 자가진단 → 예약 유도 · 긴급출동 → 관리자 확인 (약 3분)

- 경로: `/resident/safety-check` (로그인 필요 시 `/`에서 로그인 후 이동)
- 확인 포인트
  - 15문항 제출 후 결과 저장
  - 결과 카드에서 **「점검/수리 예약 바로하기」** 클릭 → `/reservation?source=diagnosis&riskScore=…` 로 이동하며 요청사항 등 초안이 채워지는지
  - **「긴급출동 요청」** 클릭 → 접수 완료 메시지
- 관리자 `/admin`에서 확인
  - 예약 목록 **우선순위** 열에 `긴급출동` 표시
  - **우선순위 필터**로 `긴급출동`만 조회
  - 예약 **CSV**에 우선순위 컬럼 포함

멘트 예시:
"자가진단 후에는 일반 예약으로 유도하고, 위험 신호가 있으면 긴급출동으로 바로 접수되며 관리자 화면에서 한눈에 구분됩니다."

### B. 예약 접수 (약 3분)

- 경로: `/reservation`
- 확인 포인트
  - 정상 입력으로 예약 등록
  - 검증 실패 케이스 확인
    - 전화번호 형식 오류
    - 과거 날짜 선택
    - 요청사항 10자 미만
  - 최근 접수 목록 로드

멘트 예시:
"클라이언트/서버 양쪽에서 입력 검증을 해서 잘못된 접수를 줄였습니다."

### C. 관리자 운영 (약 3분)

- 경로: `/admin/login` -> `/admin`
- 확인 포인트
  - 로그인/로그아웃
  - 예약 검색, 상태 필터, **우선순위(일반/긴급출동) 필터**, 정렬
  - 상태 변경 (접수 -> 진행중 -> 완료)
  - 메모 저장 및 수정시각
  - 예약 CSV 다운로드
  - 활동 로그 패널

멘트 예시:
"운영자는 검색/필터로 빠르게 찾고, 상태/메모를 업데이트하며 이력까지 확인할 수 있습니다."

### D. 백업/복원 안정성 (약 2분)

- 관리자 백업 패널에서 확인
- 확인 포인트
  - 수동 백업 생성
  - 스냅샷 미리보기
  - 복원 실행 시 체크포인트 라벨 입력
  - 복원 결과 diff 요약 확인
  - 긴급 롤백 버튼
  - 백업 JSON 다운로드

멘트 예시:
"복원 전에 체크포인트를 자동 생성하기 때문에 실수 복원도 즉시 롤백 가능합니다."

## 4) 시연 전 샘플 데이터 준비 권장

- 예약 데이터 5~10건 생성
- 상태값(접수/진행중/완료) 골고루 준비
- 메모 2~3건 이상 저장
- 체크포인트 백업 최소 1건 준비

---

## 5) 문제 발생 시 빠른 점검

- 실행 문제: `.env.local` 누락 여부 확인
- 로그인 문제: `ADMIN_PASSWORD` 값 확인
- 백업/복원 문제: `data/backups` 폴더 생성 여부 확인
- 데이터 반영 문제: 관리자 화면 새로고침(`router.refresh`) 또는 브라우저 새로고침

---

## 6) 최종 오픈 전 권장 확인

- 실제 전화번호/카카오 링크 반영
- 관리자 비밀번호 강한 값으로 변경
- 복원 절차 리허설 1회 진행
- 내부 승인 후 배포
