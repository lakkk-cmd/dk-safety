`blog_jobs`가 `pending_review`에 도달하면 대장이 `/hq/blog-jobs`에서 직접 발행한다. 자동 발행
경로는 존재하지 않는다 — 이 수동 절차 자체가 승인 게이트다.

## 절차

1. `/hq/blog-jobs`에서 `pending_review` 상태 작업을 연다 — 원고(`draft`), 보정 사진
   (`processed_images`), 썸네일(`thumbnail_url`), 검증 점수(`validation.score`)를 확인한다.
2. 문제 없으면: 원고를 네이버 블로그 에디터에 직접 붙여넣어 발행 → 발행된 글 URL을 복사 →
   화면에서 URL 입력 후 확정. 내부적으로 `publishBlogJob(id, publishedUrl)`
   (`src/lib/blog-jobs.ts`)가 `pending_review`인 행만 `published`로 전환한다(다른 상태면 에러).
3. 문제 있으면: 반려 사유를 입력하고 반려 처리 → `rejectBlogJob(id, note)`가 `rejected`로 전환.
4. 발행/반려 둘 다 자동으로 `brain/raw/external/`에 결과가 기록된다(`appendToRaw`,
   Vercel 환경이라 GitHub API 커밋 경로 사용, → [[블로그-팩토리]]).

## 확인할 것

- `published_url`은 반드시 `https://…` 형식이어야 저장됨(코드에서 강제).
- `pending_review`가 아닌 상태(`queued`/`error` 등)에서는 발행/반려 버튼이 작동하지 않음 —
  워커가 아직 처리 중이거나 이미 실패한 작업.
