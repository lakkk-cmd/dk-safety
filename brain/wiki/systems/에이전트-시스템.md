경영진 6에이전트(CTO/CSO/CMO/COO/CFO/CLO)와 3개 콘텐츠 에이전트가 `/hq` 9-에이전트 채팅으로
통합돼 있고, 별도로 "풀 에이전트"(`full-agent.ts`)가 RAG 근거표시+거짓정보 차단 채팅을 맡는다.
서버 전용 쓰기는 `AGENT_WRITE_SECRET`으로 인증한다.

## 핵심 사실

- 경영진 프롬프트: `src/lib/agents.ts`. 콘텐츠 에이전트(유튜브PD/카카오매니저/블로그에디터):
  `src/lib/content-agents.ts`.
- 9-에이전트 채팅(`/hq` 루트, `src/lib/agent-chat.ts`): `chatWithAgent()`가 `agent_chat_messages`에
  히스토리 저장, `buildBusinessSnapshot()`으로 실시간 컨텍스트 구성, RAG는 `knowledge_base`
  (`searchKnowledgeBase`, `agent-chat.ts:16`) 사용.
- 풀 에이전트(`src/lib/full-agent.ts`, `chatWithFullAgent()`): RAG는 `knowledge_chunks`
  (`searchKnowledgeChunksWithEvidence`, `full-agent.ts:38,392`) 사용 — **9-에이전트 채팅과
  다른 지식베이스를 본다.** 자세한 구분은 [[RAG-파이프라인]] 참고.
- 서버 전용 쓰기 도구(`src/lib/full-agent-tools.ts`, `/api/agent/write` 등)는 요청 헤더의
  `AGENT_WRITE_SECRET` 환경변수 값을 대조해 인증(`src/app/api/agent/write/route.ts:18`).
  이 시크릿이 GitHub Actions 시크릿에 누락되면 여러 cron이 조용히 실패한다(과거 실사례).
- 회의 스케줄: `src/lib/agent-schedule.ts` — 최초 보고 + 매주 일요일 08:00 KST 반복.
- 경영진 기억: `agent_memory` 테이블 — `structured_v1`(전략테마/결정/KPI), `content_performance_lessons`
  등 여러 용도가 한 테이블에 섞여 있음(사람이 보려면 [[content-performance-lessons]] 등 참고).
