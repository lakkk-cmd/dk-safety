Gemini 2.5 Flash가 콘텐츠/RAG답변/코드리뷰 등을 교차검증한다. 점수 기준은 용도별로 다르다 —
콘텐츠·RAG답변은 80점, 청크 관련성 판단은 50점이 통과선이다.

## 핵심 사실

- 검증 함수: `src/lib/cross-validate.ts`. `thinkingBudget: 0`으로 설정돼 있음(Gemini가
  `maxOutputTokens`를 thinking에 다 써서 응답이 잘리던 버그를 이렇게 수정, [[gemini-thinking-budget-bug]]
  참고).
- 점수 게이트: content/rag_answer 검증은 `score >= 80`이 통과 기준(`cross-validate.ts:143,190`),
  chunk_relevance(RAG 청크가 질문과 관련 있는지)는 `score >= 50`(`cross-validate.ts:233`).
- 결과는 전부 `logResult()`로 기록되어 `/admin/stats`에서 이력 조회 가능.
- 풀 에이전트는 RAG 근거 없으면 답변에 배지 표시, 거짓/위험정보 감지 시 답변 자체를 차단
  ([[full-agent-false-answer-prevention]]).
- 주제 무관성(전기안전과 관계없는 질문)은 감점 사유가 아님 — 위험한 안전 오정보일 때만 대체
  하도록 검토 기준이 조정됨([[full-agent-review-criteria-fix]]).
- 코드 변경에도 같은 Gemini 검증이 걸려 있음(`/api/code-review`) — `AGENT_WRITE_SECRET`이
  GitHub Actions 시크릿에서 누락되면 이 검증이 조용히 위장 통과했던 사고가 있었음
  ([[missing-agent-write-secret-gh]]).
