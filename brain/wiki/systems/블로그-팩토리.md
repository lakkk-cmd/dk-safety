`blog_jobs` 큐 기반 비용 0원 네이버 블로그 제작 파이프라인. 자동 발행은 네이버 정책 위반이라
만들지 않았고, **사람이 네이버 에디터에 수동으로 붙여넣어 발행하는 것 자체가 승인 게이트**다.

## 핵심 사실

- 테이블: `blog_jobs`(migration 064). 상태 흐름: `queued → researching → drafting →
  processing_images → pending_review → published`(또는 `rejected`/`error`).
- 스키마 주석에 원칙이 명시: "자동 발행은 네이버 정책 위반이라 만들지 않는다 — 사람 발행이
  곧 승인 게이트" (`supabase/migrations/064_blog_jobs.sql:4-5`).
- 발행 패키지 화면: `/hq/blog-jobs` — 대장이 여기서 완성된 원고+사진+썸네일을 받아 네이버
  에디터에 직접 붙여넣는다.
- 워커 라이브러리: `src/lib/blog-jobs.ts`. 리뷰 알림: `/api/blog-jobs/notify-review`.
- 파이프라인: 키워드 조사 → 원고 생성(RAG 참고) → 검증 → 사진 보정/썸네일 생성 →
  `pending_review`. `requested_by`는 `cmo`/`blog_editor`/`orchestrator`/`manual`.
- 네이버 검색광고 API 키가 없으면 키워드 조사는 mock 데이터 사용 — 발급 시 실데이터로 전환.
- 6단계 전부 완료, 자율 e2e 성공(2026-07-08) — 자세한 건 [[dk-blog-factory-status]] 참고.
