Solapi가 카카오 알림톡/친구톡/SMS 발송을 전담한다. 알림톡은 사전승인 템플릿이 필요하고,
친구톡은 채널 ID(PFID)만 있으면 자유 작성이 가능하다는 차이가 핵심이다.

## 핵심 사실

- 알림톡(사전승인 템플릿 필요): `SOLAPI_API_KEY` + `SOLAPI_TEMPLATE_ID` 둘 다 설정돼야 발송,
  템플릿 미설정 시 조용히 건너뜀(`src/lib/customer-notification.ts:65-67`, "템플릿 심사 대기 중"
  로그). **템플릿별 실제 승인 상태는 코드에서 확정할 수 없음 — Solapi 콘솔에서 직접 확인 필요.**
- 친구톡(채널 메시지, 사전승인 불필요): `SOLAPI_PFID`(카카오 채널 ID, `KA01PF...` 형식)만 있으면
  자유 작성 가능 — `src/lib/solapi-agent.ts:59`.
  이 PFID 값은 리포지토리 어디에도 하드코딩돼 있지 않다(env로만 주입) — 특정 값을 여기 문서에
  박제하지 않는다.
- 예약 알림/현장보고서 알림 모두 "알림톡 우선 시도 → 실패 시 SMS 폴백" 패턴
  (`customer-notification.ts`, `field-report-notification.ts`).
- E2E 수신 확인 완료(2026-06-19) — 변수 자동 변환(SDK), graceful skip 로직 포함
  (자세한 건 [[solapi-alimtalk-status]] 참고).
- 카카오 채널 OAuth("나에게 보내기" 메모 API, `kakao-publish.ts`)는 콘텐츠 승인 알림용으로
  별도 존재 — 알림톡/친구톡과는 다른 카카오 API 계열.
