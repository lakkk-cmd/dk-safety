- **Veo API 직접호출 보류** (2026-06-21): 비용 문제로 Google Veo API 직접 연동을 보류하고,
  Google Flow 수동 조작 + 마스터 캐릭터시트 + 프레임체이닝으로 영상 소스 확보 방식을 채택.
  이후 영상 팩토리는 Remotion+edge-tts 기반 자체 렌더링으로 별도 구축됨(→ [[영상-팩토리]]).

- **hq/report/agent 서브도메인 통합**: 8탭이었던 `/hq`를 홈/예약/인텔리전스/보고서 4탭으로
  정리하고, 콘텐츠/파이프라인/개선요청 탭을 각각 contents.dkansim.com/agent.dkansim.com/
  9-에이전트 채팅으로 대체. `/hq` 루트는 9-에이전트 채팅이 메인.

- **네이버 블로그 자동 발행 금지**: 네이버 정책 위반 소지가 있어 자동 발행 기능을 만들지
  않음 — 사람이 네이버 에디터에 수동으로 붙여넣어 발행하는 것 자체가 승인 게이트
  (`supabase/migrations/064_blog_jobs.sql:4-5`, → [[블로그-팩토리]]).

- **콘텐츠 승인 게이트 원칙**: 블로그/카카오/유튜브 콘텐츠는 전부 `/contents`에서 대장의
  명시적 승인을 거쳐야 발행됨(자동 발행 없음). 영상 팩토리도 동일 원칙(`pending_review`→
  `approved` 게이트, → [[영상-팩토리]]).

- **특허 표기 규칙**: 특허 등록 완료 전까지 "특허 획득"/"특허 보유" 문구 사용 금지, "특허출원번호
  기재" 방식만 사용(콘텐츠 CLO 검토에서 반복 확인, → [[회사-프로필]]).

- **dk-brain 3층 지식체계 구축** (2026-07-10): Claude Code 세션/6에이전트/콘텐츠 에이전트/
  영상·블로그 팩토리 워커가 지식을 공유하도록 `brain/raw`+`brain/wiki`+`brain/ops` 구조 도입.
  `brain/wiki`(정적 지식)만 `knowledge_base`+`knowledge_chunks` 양쪽에 임베딩하고, `brain/ops`
  (고회전 운영 데이터 미러)는 임베딩 대상에서 제외. 지원사업(grant) 팩토리는 코드에 존재하지
  않아 이번 구축 범위에서 제외.
