**결정사항**:
- dk-brain(옵시디언 3층 구조: raw/wiki/CLAUDE.md)을 repo에 도입, 기존에 만들었던 `wiki/`
  (Supabase 운영 데이터 미러)를 `brain/ops/`로 흡수 편입.
- `brain/wiki/`(정적 지식)만 임베딩하고 `brain/ops/`(고회전 운영 데이터)는 임베딩 제외 —
  이유는 오래된 데이터가 RAG를 오염시키기 때문.
- `knowledge_base`(OpenRouter, 9-에이전트 채팅/현장AI소견이 읽음)와 `knowledge_chunks`
  (Voyage, 풀 에이전트가 읽음) 양쪽에 동기화 — 원래 dk-brain 프롬프트는 하나만 지정했으나
  실제로 지식베이스가 두 개로 독립돼 있다는 걸 검증 중 발견해 둘 다로 확장.
- 지원사업(grant) 팩토리는 코드에 존재하지 않아 이번 구축 범위에서 제외.

**변경된 파일**:
- 신규: `brain/README.md`, `brain/raw/decisions/2026-확정-기술결정.md`,
  `brain/wiki/systems/*.md`(9개), `brain/wiki/profile/*.md`(2개),
  `brain/wiki/lessons/*.md`(32개, 기존 Claude Code 세션 메모리 이관),
  `scripts/brain-sync.ts`, `worker/lib/append-to-raw.mjs`, `src/lib/append-to-raw.ts`.
- 이동: `wiki/` → `brain/ops/` (`scripts/sync-wiki-to-git.mjs` 경로 갱신,
  npm script `wiki:sync` → `brain:sync:ops`).
- 수정: `CLAUDE.md`(지식 체계 규칙 섹션 추가), `src/lib/blog-jobs.ts`/`src/lib/video-jobs.ts`
  (발행/반려 시 `appendToRaw()` 호출), `worker/index.mjs`(영상/블로그 워커 종료 시
  `appendToRaw()` 호출), `package.json`(`brain:sync`, `brain:sync:ops` 스크립트).

**알게 된 것**:
- dk-brain 프롬프트의 특허출원번호(`10-2026-0082817`)가 실제 코드 값(`4-2026-034423-8`,
  `scripts/seed-knowledge-base.ts`)과 달랐음 — 검증 없이 실행했으면 잘못된 값이 영구
  문서화될 뻔함.
- `knowledge_base`와 `knowledge_chunks`는 완전히 독립된 지식베이스(임베딩 제공자도 다름)이고,
  각각 다른 채팅 시스템이 읽는다는 게 코드 확인 전엔 명확하지 않았음.
- Vercel 서버리스 함수는 로컬 파일쓰기가 영구 저장되지 않으므로, `blog-jobs.ts`/`video-jobs.ts`
  (Vercel에서 실행)의 `appendToRaw`는 GitHub API 커밋 경로(`writeGithubFile`, 기존 자가개선
  파이프라인과 동일 인증)를 쓰고, `worker/`(로컬 PC 실행)의 `appendToRaw`는 일반 `fs` 쓰기를
  쓴다 — 실행 환경에 따라 구현을 분리해야 했음.
- KEC 210/212/232/234 조항은 실제로 `scripts/seed-regulation-knowledge.mjs`가 시드한 값으로
  확인됨(dk-brain 프롬프트의 해당 부분은 정확했음).

**미해결**:
- `OPENROUTER_API_KEY`가 로컬 `.env.local`에 없어 `knowledge_base` 쪽 동기화는 이번 세션에서
  실제로 실행/검증하지 못했음(코드는 기존 `seed-knowledge-base.ts`와 동일한 함수를 재사용해
  구현했지만, 프로덕션 키로 실제 실행 검증 필요).
- 사업자등록번호 208-20-57629, 일반과세자 여부는 여전히 미검증(`brain/wiki/profile/회사-프로필.md`에
  명시).
- `brain/wiki/playbooks/`는 스켈레톤만 있고 아직 내용 없음 — 반복 업무 매뉴얼은 다음 세션 과제.
