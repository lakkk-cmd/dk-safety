**결정사항**:
- ReRoomAI(외부 데모 앱, aicitybuilders.com 소개) 검토 후 3개 기법 이식: 점검 전/후 비교
  슬라이더, 사진 업로드 클라이언트단 다운스케일, 공개 API IP 레이트리밋.
- Gemini `gemini-3.1-flash-image-preview`로 영상 팩토리 이미지 생성(Flux) 교체 — **보류**.
  무료 티어 없음(전체 비용 0원 요건 불충족), preview 모델 rate limit/폐지 리스크,
  GEMINI_API_KEY 공유로 인한 실시간 채팅 quota 경합 위험, 애초에 구조 보존 편집을 적용할
  실사진 입력이 지금 파이프라인에 없음(순수 배경 생성용) — 4가지 이유로 기각.
- "실제 현장사진을 Gemini로 보정" 신규 기능도 검토 후 보류 — 사진 보정은 이미 sharp로 무료
  처리 중(`worker/lib/image-process.mjs`), Gemini는 sharp가 못 하는 콘텐츠 변경만 추가하는데
  그건 정직성 이슈가 다시 걸림 + 점검 증빙 사진에는 애초에 부적절.
- 외부 스킬 6개(카파시 가이드라인/claude-video/superpowers/Understand-Anything/
  agentmemory/한국어 humanizer) 검토 → **humanizer만 설치**. 나머지는 기존 시스템(oh-my-claudecode,
  dk-brain, CLAUDE.md)과 중복이거나(karpathy-guidelines, Understand-Anything), 자동 활성화라
  충돌 위험이 있거나(superpowers), dk-brain과 정확히 같은 문제를 푸는 경쟁 시스템이라
  (agentmemory) 기각.
- humanizer를 Claude Code 세션용으로만 두지 않고, **실제 프로덕션 콘텐츠 파이프라인**
  (`content-pipeline.ts`)에 연결 — 유튜브/카카오/블로그 초안이 Gemini 검증(및 카카오/블로그는
  자동발행) 이전에 자동 교정되도록 구현.

**알게 된 것**:
- Gemini 이미지 모델 가격은 이미지당이 아니라 입력은 토큰 100만개당, 출력은 해상도별
  (0.5K $0.045 ~ 4K $0.151) — 처음 요약에서 "$0.50/이미지"로 오인할 뻔했으나 재조회로 정정.
- `next.config.ts`의 `remotePatterns`에 Supabase 도메인이 없어서, Supabase Storage에 있는
  이미지는 `next/image`가 아니라 일반 `<img>`를 써야 함(비교 슬라이더 만들 때 발견).
- `field-report-wizard.tsx`/`worker-task-detail.tsx` 둘 다 사진을 원본 그대로 업로드하고
  있었음(클라이언트 압축 전무) — 다운스케일 추가로 해소.
- `content-pipeline.ts`의 블로그·카카오는 **Gemini 검증 통과 시 사람 승인 없이 즉시 발행**되는
  구조임을 코드 주석에서 확인 — CLAUDE.md의 "human-in-the-loop" 서술과 실제 코드 동작이
  어긋나 있음(운영 리스크로 별도 기록, → raw/decisions 참고).

**미해결**:
- humanizer 파이프라인 연결로 Claude API 호출이 콘텐츠 1건당 1회 추가됨 — 실제 월간 비용
  증가폭은 아직 측정 안 함.
- `brain/wiki/systems/`에 이번에 추가한 4개 유틸리티(비교슬라이더/다운스케일/레이트리밋/
  휴머나이저)를 문서화 안 함 — 이 세션 로그 작성과 함께 보완 필요(같은 세션에서 이어서 처리).
