# 대경안심전기 내부 시연/테스트 가이드 (출력용)

문서 목적: 내부 시연/검증 및 오픈 전 승인 체크  
프로젝트: 대경안심전기 (`우리 집 전기 주치의`)  
버전: 내부 시연본 v1

---

## 1. 실행 준비

- `.env.example` -> `.env.local` 복사
- 값 입력
  - `NEXT_PUBLIC_BUSINESS_PHONE`
  - `NEXT_PUBLIC_KAKAO_OPENCHAT_URL`
  - `ADMIN_PASSWORD`
- 실행
  - `npm install`
  - `npm run dev`
- 접속: `http://localhost:3000`

---

## 2. 시연 시나리오 (10분)

### 2.1 입주민 로그인 + 홈 (2분)

- 경로: `/` (로그인) → `/home` (서비스 메인)
- 확인 항목
  - 브랜드 메시지 노출
  - 로그인 후 홈: 서비스 카드, 예약·자가진단 CTA
  - 전화/카카오 CTA(퀵액션 포함)

### 2.1b (선택) 자가진단 → 예약 · 긴급출동 → 관리자 (3분)

- 경로: `/resident/safety-check` (미로그인 시 `/` 로그인 후)
- 확인 항목
  - 15문항 제출·결과 저장
- 「점검/수리 예약 바로하기」→ `/reservation?source=diagnosis&riskScore=…` 초안 반영
  - 「긴급출동 요청」→ 접수 완료
- `/admin`: 우선순위 열·`긴급출동` 필터, 예약 CSV 우선순위 컬럼

### 2.2 예약 접수 (3분)

- 경로: `/reservation`
- 확인 항목
  - 정상 접수
  - 유효성 검사 실패 케이스
    - 전화번호 형식
    - 과거 날짜
    - 짧은 요청사항

### 2.3 관리자 운영 (3분)

- 경로: `/admin/login` -> `/admin`
- 확인 항목
  - 로그인/로그아웃
  - 검색/상태·우선순위 필터/정렬/페이지네이션
  - 상태 변경
  - 메모 저장 + 수정시각
  - CSV 다운로드
  - 활동 로그 조회

### 2.4 백업/복원 안정성 (2분)

- 확인 항목
  - 수동 백업 생성
  - 스냅샷 미리보기
  - 복원(체크포인트 라벨 입력)
  - 복원 결과 diff 요약
  - 긴급 롤백
  - 백업 JSON 다운로드

## 4. 문제 발생 시 점검

- 실행 불가: `.env.local` 존재/값 확인
- 관리자 로그인 실패: `ADMIN_PASSWORD` 확인
- 백업 오류: `data/backups` 생성 여부 확인
- 화면 미반영: 브라우저 새로고침 또는 관리자 페이지 재진입

---

## 5. 오픈 승인 기준

- 필수 기능 100% 통과
- 복원/롤백 리허설 1회 통과
- 실연락처/실카카오 링크 반영
- 관리자 비밀번호 운영용으로 교체
